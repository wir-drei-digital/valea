# Mail (Phase 4) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A sync-to-files IMAP engine that lands the user's `AI/Review` mail as markdown in `sources/mail/`, closes the approval loop back into the mailbox (APPEND draft, MOVE to Processed), and replaces the mock input — per `docs/superpowers/specs/2026-07-11-mail-design.md` (v2, post-external-review; READ IT for any ambiguity).

**Architecture:** All engine code in the Elixir backend under `Valea.Mail.*`: a vendored minimal IMAP client (`ImapClient`) behind a `Transport` behaviour, a `Normalizer` over gen_smtp's `:mimemail`, and a per-workspace `Engine` GenServer under `Valea.Workspace.Runtime` that activates only after migration. Credentials come from the OS keychain via two Tauri commands and live in Engine RAM only. The queue envelope grows to `queue_item/v2` with crash-durable `mailbox_ops` intents.

**Tech Stack:** Elixir 1.15+/OTP, Phoenix 1.8, Ash 3 + AshSqlite (first real tables), gen_smtp (`:mimemail`), Floki, Codepagex, SvelteKit/Svelte 5 runes, Tauri v2 + `keyring` crate.

## Global Constraints

Binding on every task; the spec is the source of truth when in doubt.

- **No credentials on disk, ever.** Not in the workspace, not in config files, not in logs/audit/doctor output/error messages. Engine holds the secret in process memory only.
- **TLS is mandatory**: implicit TLS on the configured port (default 993), `verify: :verify_peer` against `:public_key.cacerts_get()`, hostname verification, SNI. No `ssl:` toggle anywhere. (Tests may inject a fixture CA via a documented `:tls_opts` override — never weakening `verify_peer`.)
- **A bare `EXPUNGE` is never issued.** Move ladder: `UID MOVE` → (UIDPLUS) `UID COPY` + `UID STORE +FLAGS (\Deleted)` + `UID EXPUNGE <uid>` → `{:unsupported, _}`.
- **UIDs only** (never sequence numbers); **`BODY.PEEK[...]`** for all fetches; **`RFC822.SIZE` checked before** any full-body fetch.
- **Files are canonical, SQLite is cache**: every table must be rebuildable from `sources/mail/` + resync; wiping `app.sqlite` must never lose data.
- **Migration never deletes or overwrites a user-modified file.** Pristine detection is by SHA-256 against the shipped v2 seed bytes; legacy files are archived to `logs/migrations/v3/`, never deleted.
- **Message file paths are stable forever**: server-side moves flip `status:` in frontmatter; the file never moves or vanishes.
- **Frontmatter injection hardening**: every value interpolated into YAML frontmatter rejects C0/DEL control chars (mirror `Valea.Queue.no_control_chars?/1`) and quotes/escapes strings.
- **The approval flow's Phase-3 ordering is inviolable** (`Valea.Queue.approve/2` steps 1–7); Phase 4 only *extends* step 6 (intent rewrite before terminal rename) and adds post-approval work that can never block or undo an approval.
- **The managed `.claude/settings.json` deny rules are unchanged**; nothing in this phase adds agent-readable secrets.
- Backend: `mix format` clean, `mix test` green, no compiler warnings. Frontend: `bun run check` (svelte-check) 0 errors, `bun run test` green. Desktop: `cargo check` clean.
- Commits end with trailer: `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`. Never push to origin.
- Run backend tests from `backend/`, frontend from `frontend/`, cargo from `desktop/src-tauri/`.

### Task 1: IMAP wire codec (`Valea.Mail.Imap.Wire`)

Pure functions, no sockets. This is where literal-exact parsing lives — the single most bug-prone part of IMAP; regex-over-text is forbidden.

**Files:**
- Create: `backend/lib/valea/mail/imap/wire.ex`
- Test: `backend/test/valea/mail/imap/wire_test.exs`

**Interfaces (Produces):**
```elixir
# Pull exactly one complete server response off the front of `buffer`.
@spec pull(binary()) :: {:ok, response(), rest :: binary()} | :incomplete
# response() is one of:
#   {:tagged, tag :: binary, :ok | :no | :bad, text :: binary}
#   {:untagged, line :: binary}                       # non-FETCH untagged, raw line (no CRLF)
#   {:fetch, seq :: integer, attrs :: map}            # parsed FETCH: %{uid: int|nil, size: int|nil,
#                                                     #   flags: [binary], body: binary|nil, header: binary|nil,
#                                                     #   internaldate: binary|nil}
#   {:continuation, text :: binary}                   # "+ ..." (for APPEND/literals)

# Encode one command; literals become {:literal, bin} parts using
# LITERAL+ synchronizing form is NOT used — plain literal with continuation.
@spec encode(tag :: binary, parts :: [binary | {:literal, binary}]) ::
        {iodata, pending_literals :: [binary]}
# encode("A1", ["APPEND", "Drafts", "(\\Draft)", {:literal, rfc822}]) returns
# {"A1 APPEND \"Drafts\" (\\Draft) {123}\r\n", [rfc822]} — the caller sends the
# first iodata, waits for {:continuation, _}, sends the literal + "\r\n".
```

Parsing rules (implement exactly):
- A response line ends at CRLF **unless** it contains `{N}\r\n` — then N bytes of literal follow, then the line continues until the next CRLF (repeat for multiple literals).
- `pull/1` returns `:incomplete` whenever the buffer holds less than a full response (including a partially received literal). It never guesses.
- FETCH lines look like `* 12 FETCH (UID 4711 RFC822.SIZE 2311 FLAGS (\Seen) BODY[] {2311}\r\n<bytes>)`. Parse the parenthesized attr list with a small recursive tokenizer: atoms, quoted strings, parenthesized lists, and literals. Map `BODY[]`→`body`, `BODY[HEADER.FIELDS (...)]`→`header`, `UID`→`uid` (integer), `RFC822.SIZE`→`size` (integer), `FLAGS`→`flags` (list of flag strings), `INTERNALDATE`→`internaldate`.
- Tagged lines: `A3 OK ...` / `A3 NO ...` / `A3 BAD ...`.
- Argument quoting in `encode/2`: an argument goes unquoted if it matches `^[A-Za-z0-9\\().+*:\[\]-]+$` or starts with `(`, otherwise it is a quoted string with `\` and `"` escaped; anything containing CR/LF/8-bit must be a `{:literal, _}` (raise `ArgumentError` otherwise — command injection guard).

defmodule Valea.Mail.Imap.WireTest do
  use ExUnit.Case, async: true
  alias Valea.Mail.Imap.Wire

  test "pulls a tagged OK" do
    assert {:ok, {:tagged, "A1", :ok, "LOGIN completed"}, ""} =
             Wire.pull("A1 OK LOGIN completed\r\n")
  end

  test "incomplete without CRLF" do
    assert :incomplete = Wire.pull("A1 OK LOGIN comp")
  end

  test "fetch with literal body parses by exact byte count" do
    body = "From: a@b\r\n\r\nhello)world"  # contains ')' — regex would break here
    line = "* 2 FETCH (UID 7 RFC822.SIZE #{byte_size(body)} BODY[] {#{byte_size(body)}}\r\n#{body})\r\n"
    assert {:ok, {:fetch, 2, attrs}, ""} = Wire.pull(line <> "")
    assert attrs.uid == 7
    assert attrs.body == body
  end

  test "incomplete mid-literal" do
    assert :incomplete = Wire.pull("* 2 FETCH (BODY[] {10}\r\nabc")
  end

  test "untagged capability line passes through raw" do
    assert {:ok, {:untagged, "CAPABILITY IMAP4rev1 MOVE UIDPLUS"}, ""} =
             Wire.pull("* CAPABILITY IMAP4rev1 MOVE UIDPLUS\r\n")
  end

  end

  test "two responses pull one at a time" do

    assert {:ok, {:tagged, "A2", :ok, "done"}, ""} = Wire.pull(rest)
  end

  test "encode quotes folder names with spaces" do
    {iodata, []} = Wire.encode("A1", ["SELECT", "AI/Review Folder"])

    assert lit == "abc"
  end

  test "encode raises on CR/LF smuggled into a bare argument" do
    assert_raise ArgumentError, fn -> Wire.encode("A3", ["SELECT", "x\r\nA4 DELETE INBOX"]) end
  end

defmodule Valea.Mail.Transport do
  @type conn :: term()
  @type config :: %{host: String.t(), port: pos_integer(), username: String.t()}
  @callback connect(config, credential :: String.t(), opts :: keyword()) ::
              {:ok, conn} | {:error, term()}
  @callback capabilities(conn) :: {:ok, [String.t()]}
  @callback list_folders(conn) :: {:ok, [String.t()]}
  @callback create_folder(conn, String.t()) :: :ok | {:error, term()}
  @callback select(conn, String.t()) :: {:ok, %{uidvalidity: integer(), uidnext: integer() | nil}} | {:error, term()}
  @callback uid_search(conn, String.t()) :: {:ok, [pos_integer()]} | {:error, term()}
  @callback uid_fetch_meta(conn, [pos_integer()]) :: {:ok, [%{uid: pos_integer(), size: non_neg_integer()}]} | {:error, term()}
  @callback uid_fetch_headers(conn, [pos_integer()]) :: {:ok, [%{uid: pos_integer(), header: binary()}]} | {:error, term()}
  @callback uid_fetch_full(conn, pos_integer()) :: {:ok, binary()} | {:error, term()}
  @callback uid_move(conn, pos_integer(), String.t()) :: :ok | {:error, term()} | {:unsupported, String.t()}
  @callback append(conn, folder :: String.t(), flags :: [String.t()], rfc822 :: binary()) :: :ok | {:error, term()}
  @callback logout(conn) :: :ok
end
```

ImapClient specifics:
- `connect/3`: `:ssl.connect(host, port, tls_opts ++ [active: false, mode: :binary, packet: :raw])`. Default `tls_opts`: `[verify: :verify_peer, cacerts: :public_key.cacerts_get(), server_name_indication: to_charlist(host), customize_hostname_check: [match_fun: :public_key.pkix_verify_hostname_match_fun(:https)], depth: 3]`. `opts[:tls_opts]` (keyword) is **merged over** the defaults — used by tests to supply `cacertfile:`; a moduledoc note states it must never be used to disable `verify_peer` outside tests. Read the greeting, LOGIN, then refresh CAPABILITY (post-login). `conn` = `%ImapClient.Conn{socket:, capabilities: MapSet, tag: counter}`.
- Auth failure (tagged `NO` on LOGIN) returns `{:error, :auth_failed}` — the Engine keys off this exact atom.
- `uid_move/3` ladder: `MOVE` capability → `UID MOVE`; else `UIDPLUS` → `UID COPY` + `UID STORE +FLAGS (\Deleted)` + `UID EXPUNGE <uid>`; else `{:unsupported, "server has neither MOVE nor UIDPLUS"}`. **Grep-able invariant: the string `"EXPUNGE"` must only ever be encoded as `"UID EXPUNGE"`.**
- `append/4`: encode with `{:literal, rfc822}`, wait for `{:continuation, _}`, send literal + CRLF, await tagged OK.
- `uid_search/2` parses `* SEARCH 4 7 9`; `select/2` parses `* OK [UIDVALIDITY n]` / `* OK [UIDNEXT n]` from untagged lines.
- Every call reads socket data into a buffer and drains via `Wire.pull/1` until the command's tagged response arrives; 30s recv timeout → `{:error, :timeout}`.

Normalization rules (from spec §Normalizer — implement all):
- `:mimemail.decode/1` inside a `try/rescue`; on raise → best-effort: headers via `parse_headers/1`, body = raw text after the first blank line (validated UTF-8 or replaced), `notes.normalizer_note` set. **`normalize/1` never returns an error for malformed input.**
- Body: first `text/plain` part depth-first; else first `text/html` through Floki (`Floki.parse_document!` → drop `script`/`style` → block elements (`p, div, br, li, tr, h1..h6`) become newlines → `Floki.text`), collapse 3+ newlines to 2.
- Charsets: decode `quoted-printable`/`base64` transfer encodings (mimemail does this); charset param `utf-8`/`us-ascii` → validate with `String.valid?/1`; `iso-8859-1`/`latin1`/`windows-1252`/`cp1252` → `Codepagex.to_string!(bytes, :iso_8859_1)` (map cp1252 to `"VENDORS/MICSFT/WINDOWS/CP1252"`); anything else → `:unicode` replacement scrub + `notes.charset_note`. **Configure codepagex encodings in `backend/config/config.exs`: `config :codepagex, :encodings, [:iso_8859_1, "VENDORS/MICSFT/WINDOWS/CP1252"]`.**
- Encoded-words in Subject/From (`=?utf-8?B?...?=`): mimemail decodes these in headers when using `:mimemail.decode`; for `parse_headers/1` (headers-only path) implement RFC 2047 decode for B/Q encodings over the same charset table.
- Attachments: any part with `Content-Disposition: attachment` or a filename param. `sanitize_filename/1`: `Path.basename`, strip C0/DEL + `/ \\ :`, collapse to `"attachment"` if empty, prepend nothing.
- `render/2`: frontmatter exactly per spec example (id, message_id, from, to, subject, date ISO8601, uid, in_reply_to, references, reply_to, status, source, source_ref, attachments, then any notes). YAML strings double-quoted with `\\` and `"` escaped; reject C0/DEL control chars in header-derived values by replacing with `" "` (a mail header must never break the frontmatter). Body below `---`, verbatim.
- `flip_status/2`: parse the leading `---\n...\n---\n` block, replace the `status:` line only, byte-preserve everything else.

// keychain.rs — service = bundle identifier; account = "<workspace_id>:<username>".
// Never log the secret. Errors map to a short string for the frontend.
use keyring::Entry;

const SERVICE: &str = "digital.wirdrei.valea";

fn entry(workspace_id: &str, username: &str) -> Result<Entry, String> {
    if workspace_id.is_empty() || username.is_empty() {
        return Err("bad key".into());
    }
    Entry::new(SERVICE, &format!("{workspace_id}:{username}")).map_err(|e| e.to_string())
}

#[tauri::command]
pub fn mail_secret_set(workspace_id: String, username: String, secret: String) -> Result<(), String> {
    entry(&workspace_id, &username)?.set_password(&secret).map_err(|e| e.to_string())
}

#[tauri::command]
pub fn mail_secret_get(workspace_id: String, username: String) -> Result<Option<String>, String> {
    match entry(&workspace_id, &username)?.get_password() {
        Ok(s) => Ok(Some(s)),
        Err(keyring::Error::NoEntry) => Ok(None),
        Err(e) => Err(e.to_string()),
    }
}

#[tauri::command]
pub fn mail_secret_delete(workspace_id: String, username: String) -> Result<(), String> {
    match entry(&workspace_id, &username)?.delete_credential() {
        Ok(()) | Err(keyring::Error::NoEntry) => Ok(()),
        Err(e) => Err(e.to_string()),
    }
}
```
